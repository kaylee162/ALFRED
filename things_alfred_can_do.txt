Updated Error handling

Bad requests and how they are handled:
Empty input - "Tell me what you need me to do."
Ollama is not running - "I couldn't connect to Ollama. Make sure Ollama is running, then try again."
Backend is off - "I couldn't reach the backend."
Backend returns a bad JSON - "The backend returned something ALFRED could not read."
Tool name is missing - "I understand this needs a tool, but I couldn't figure out which one."
Calendar API fails - "I understood the request, but the calendar tool failed."
Project folder path is invalid- Tool should return a friendly message instead of crashing
Weather API fails - Tool should return something like "I couldn’t get the weather right now."
“delete the event” with no matching events - Calendar tool should say no matching event was found
“update my meeting” with multiple matches - Calendar tool should show options instead of guessing
“create an event tomorrow” with no title - Should show missing title form
“open project fake-folder-name” - Should say project not found or show project explorer
Random unclear request like “do the thing” - Falls back to chat or asks for clarification
Invalid local file action - Should say it cannot access or find that file
